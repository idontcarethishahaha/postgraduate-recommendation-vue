## 研究生推免系统-前端

该模块是研究生推免系统的前端实现，基于**现代Vue 3生态栈**构建，适配高校研究生推免场景的多角色后台管理需求，支持管理员、学院管理员、辅导员、学生等角色的权限隔离与业务操作，提供高效、规范的前端交互体验。

## 核心技术栈
| 技术分类         | 技术名称 & 核心用途                                                                 |
|------------------|-------------------------------------------------------------------------------------|

## 核心特性
### 1. 多角色权限管控
- 基于Vue Router导航守卫（`router.beforeEach`）+ `sessionStorage`存储的角色信息，实现不同角色（管理员/学院管理员/学生/辅导员）的路由拦截；
- 动态渲染菜单与操作按钮，严格隔离不同角色的功能可见性，防止越权操作。

### 2. 标准化接口交互
- Axios拦截器自动注入Token到请求头，统一处理响应码、数据反序列化与异常提示；
- 基于TanStack Vue Query封装`query`（查询）/`mutation`（增删改），统一管理请求加载态、错误态与缓存，减少重复代码。

### 3. 高效的表单与数据处理
- 结合Element Plus表单组件实现自定义校验逻辑（学生注册、学院/专业配置等场景）；
- 支持多层嵌套路由设计（如`/admin/colleges`/`/collegeadmin/major-categories/:collegeId`），适配后台系统层级化业务。

### 4. 工程化与可维护性
- 基于Vite的快速构建与热更新，提升开发效率；
- ESLint + Prettier统一代码风格，vue-tsc保障TypeScript类型校验全覆盖；
- 接口代理配置解决跨域问题，适配前后端分离开发模式。

## 核心功能
- 角色权限：不同角色的菜单/页面/按钮权限控制；
- 基础配置：学院、专业、推免方向等基础信息管理；
- 推免流程：学生信息填报、审核进度跟踪、审批操作；
- 数据展示：推免数据统计、表格分页/筛选、表单校验；
- 登录认证：JWT令牌登录、无感刷新、密码安全校验

## 技术亮点
- 贴合2024年Vue生态主流实践（Vue 3 + Vite + Pinia + TS），适配企业级中后台开发规范；
- 异步请求规范化处理，兼顾性能（缓存）与用户体验（加载态/错误态）；
- 权限体系前端全链路管控，从路由到按钮层层隔离，保障系统安全。
